# Техническое задание: Система управления проектом с использованием pathlib

## 1. Описание задачи

Разработать утилиту для управления структурой проекта, которая:
- Сканирует директорию проекта и анализирует его структуру
- Находит все исходные файлы (Python, JavaScript, Markdown и т.д.)
- Генерирует отчет о структуре проекта
- Позволяет выполнять операции с файлами (копирование, удаление, переименование)
- Работает кроссплатформенно (Windows, Linux, macOS)

Реальный сценарий: разработчик хочет быстро понять структуру незнакомого проекта, найти все конфигурационные файлы, скопировать нужные файлы в другую директорию или очистить временные файлы.

## 2. Цель работы

Создать практический пример использования `pathlib` для:
- Навигации по файловой системе
- Поиска файлов по расширению и паттернам
- Анализа структуры проекта
- Выполнения операций с файлами
- Демонстрации кроссплатформенности

## 3. Функциональные требования

### 3.1 Сканирование проекта
- Рекурсивный поиск всех файлов в директории
- Группировка файлов по типам (исходный код, конфиги, документация, тесты)
- Подсчет количества файлов каждого типа
- Вычисление общего размера файлов

### 3.2 Анализ структуры
- Определение глубины директорий
- Поиск пустых папок
- Выявление больших файлов (> 1 MB)
- Поиск дублирующихся имен файлов

### 3.3 Поиск файлов
- Поиск по расширению (`.py`, `.js`, `.md` и т.д.)
- Поиск по паттерну (например, все файлы, начинающиеся с `test_`)
- Поиск по размеру (больше/меньше N байт)
- Поиск по дате изменения

### 3.4 Операции с файлами
- Копирование файлов с сохранением структуры
- Удаление файлов по паттерну
- Переименование файлов массово
- Создание резервной копии директории

### 3.5 Отчетность
- Вывод структуры проекта в виде дерева
- Генерация текстового отчета
- Экспорт результатов в JSON

## 4. Нефункциональные требования

- **Производительность**: обработка проектов с тысячами файлов за < 5 секунд
- **Удобство**: интуитивный интерфейс командной строки
- **Безопасность**: подтверждение перед удалением файлов
- **Кроссплатформенность**: работа на Windows, Linux, macOS без изменений
- **Читаемость**: код должен демонстрировать лучшие практики использования `pathlib`

## 5. Ограничения и допущения

- **Не работает с сетевыми путями**: только локальная файловая система
- **Не работает с символическими ссылками**: игнорируются при сканировании
- **Максимальная глубина**: ограничение на глубину рекурсии (по умолчанию 20 уровней)
- **Размер файлов**: работает с файлами до 2 GB
- **Права доступа**: требуются права на чтение файлов в директории
- **Исключения**: игнорируются скрытые файлы (начинающиеся с `.`) по умолчанию

## 6. Источник ТЗ

Самостоятельная постановка на основе типичных задач разработчика при работе с проектами. Задача охватывает основные операции, которые выполняются с файловой системой в реальных проектах.
